API ENDPOINTS REQUIRED
================================================================================

PERSONS ENDPOINTS:
- GET /api/persons - List all persons
- POST /api/persons - Create person
- GET /api/persons/<id> - Get person details
- PUT /api/persons/<id> - Update person
- DELETE /api/persons/<id> - Delete person

INTERACTIONS ENDPOINTS:
- GET /api/interactions - List interactions
- POST /api/interactions - Create interaction

REPORTS ENDPOINTS:
- POST /api/reports/intervention/<id> - Generate report
- POST /api/cases/escalate/<id> - Escalate case

WEBSOCKET ENDPOINTS:
- WS /ws/persons - Person update stream
- WS /ws/interactions - Interaction update stream

See PERSON_MANAGEMENT_BACKEND.md for complete specifications.
